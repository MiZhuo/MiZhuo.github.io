SpringBoot整合ELFK实现查看日志自由

发表于2022-07-08|更新于2022-10-09|JAVA
|字数总计:2.4k|阅读时长:12分钟|阅读量:

之前讲过如何使用Docker安装ELK,本章来讲讲如何使用SpringBoot整合ELK以及ELFK,从而实现日志自由,再也不用跑服务器上捞日志了。

搭建ELK

如果还没搭建好ELK的,可以看看之前的博客。传送门:🚪🚪🚪

Logback

要使用logstash收集SpringBoot的日志,主要是通过Logback配置来实现。

  • 添加依赖,我这里用的是gradle工程,maven的话可以拿下面的坐标自行转换。

  • 在resources下新建logback-spring.xml配置文件。

    注意:此处不能命名为logback.xml,否则logback无法加载到application.properties里面的东西,因为加载顺序是这样的:logback.xml—>application.properties—>logback-spring.xml

  • 配置logback-spring.xml

  • 配置application.yml

  • 配置logstash.conf

  • 打开kibana控制面板

    可以看到已经创建了索引,对应的就是logstash配置的索引名称。

  • 添加索引到kibana的索引模式

  • 进入Discover


    可以看到日志已经全部可以在kibana上查看了，并且可以根据其中关键字查询。

​ELK已经搭建好了，但是有个弊端就是SpringBoot运用logstash-logbak-encoder将日志发送给logstash，相当耗费服务器资源，会对CPU，内存造成一个相当高的损耗。因此我们需要对我们的架构进行升级。

FileBeat

FileBeat，它是一个轻量级的日志收集处理工具(Agent)，Filebeat占用资源少，适合于在各个服务器上搜集日志后传输给Logstash，官方也推荐此工具。

  • 新建filebeat.yml

  • 修改logstash.conf

  • 修改logback-spring.xml

  • 将SpringBoot应用部署到docker
    映射日志目录到宿主机

    docker run --name demo -v /root/logs:/home/logs -p 8888:8888 d 镜像ID

  • 启动filebeat
    将日志目录映射至filebeat，这样filebeat就能读到springboot的日志文件了。

  • 查看kibana

​至此,ELFK已经搭建好了。后续可以再将kafka接入进来，进一步提高日志收集器的性能。
