父类静态代码块->子类静态代码块->父类代码块->父类构造->子类代码块->子类构造->子类方法

JAVA动态代理的方式

  • JDK动态代理
    • 利用拦截器（必须实现InvocationHandler）加上反射机制生成一个代理接口的匿名类，在调用具体方法前调用InvokeHandler来处理
  • cglib动态代理
    • 利用ASM框架，对代理对象类生成的class文件加载进来，通过修改其字节码生成子类来处理
      https://www.cnblogs.com/sandaman2019/p/12636727.html

什么时候用cglib,什么时候用jdk动态代理？

  • 目标对象生成了接口 默认用JDK动态代理
  • 如果目标对象使用了接口，可以强制使用cglib
  • 如果目标对象没有实现接口，必须采用cglib库，Spring会自动在JDK动态代理和cglib之间转换

装箱和拆箱的原理

  • 只有double和float的自动装箱代码没有使用缓存，每次都是new 新的对象，其它的6种基本类型都使用了缓存策略。
  • 使用缓存策略是因为，缓存的这些对象都是经常使用到的（如字符、-128至127之间的数字），防止每次自动装箱都创建一次对象的实例。而double、float是浮点型的，没有特别的热的（经常使用到的）数据的，缓存效果没有其它几种类型使用效率高。

Array.asList是否可以扩容

  • Array.asList 返回的是一个 java.util.Arrays.ArrayList,是Arrays的一个静态内部类，它没有完全实现List的方法
  • Arrays.ArrayList是一个定长集合，因为它没有重写add,remove方法，所以一旦初始化元素后，集合的size就是不可变的。

泛型 T 与 ？的区别

  • T 代表某一类具体的对象，list集合里只能存放同一类型数据，如果插入不同类型数据则会报错。
  • ？可以表示成占位符，它自己也不知道list集合中会存放多少种类型的数据，所以这样就表明我们的list中存放N种数据类型也是可以的。

List、Map、Set异同点，底层方法

  • List
    • ArrayList:查询修改块，添加删除慢，因为需要移动数据。
    • Vector:线程同步，效率较低。
    • LinkedList用的链表结构存储数据，插入删除较快。
  • Set
    • HashSet:无序
  • Map
    • HashMap:数组+链表+红黑树
    • ConcurrentHashMap:1.7使用分段锁，1.8使用CAS+Synchronized

HashMap什么时候重新hashCode方法和equals方法

HashMap 是不允许有重复元素的，其实现是通过判断对象的hashcode值是和equals方法,如果不重新hashCode方法和equals方法，则会判定这两个对象不是同一个,则HashMap里面可以存放两个属性相同的实例,重写hashcode方法和equals方法后就判定两个是同一个对象，就不会存放两个相同的实例了。

事务失效有哪些场景

  • 数据库不支持事物,像MySQL的MyISAM引擎是不支持事务的。
  • 数据源未配置事务管理器
  • @Transaction 可以用在类上、接口上、public方法上,如果用在了非public方法上,事务将无效。
  • 异常被捕获了,则不会回滚。
  • 抛出的异常不是RuntimeException和Error,则不会回滚。可以自定义什么异常进行回滚。
  • spring是通过aop的方式，对需要spring管理事务的bean生成了代理对象，然后通过代理对象拦截了目标方法的执行，在方法前后添加了事务的功能，所以必须通过代理对象调用目标方法的时候，事务才会起效。

Synchronized为什么线程安全

  • 修饰代码块
    使用monitor enter 和 monitor exit 指令,指向代码起始位置及结束位置,线程尝试获取锁,当计数器为0则获取成功,计数器加1,直到代码块执行完毕,计数器减1,计数器为0,其它线程才可以去获取锁。
  • 修饰方法
    标识该方法为同步方法。JVM 根据标识来判断是否为同步方法,从而执行同步调用。

锁的状态，升级过程

锁的状态总共有四种，级别由低到高依次为：无锁、偏向锁、轻量级锁、重量级锁。

  • 偏向锁:当第一次执行到同步代码块时,锁对象会变成偏向锁,也就是偏向第一个获得它的线程,代码执行完后,线程不会主动释放偏向锁,当第二次执行到同步代码块时,线程会先判断此时持锁的线程是否是自己,如果是则直接进入,如果自始至终使用锁的线程只有一个，很明显偏向锁几乎没有额外开销，性能极高。
  • 轻量锁:当偏向锁被其它线程访问,偏向锁会升级为轻量锁,其它线程会自旋等待锁。线程不会阻塞以提高性能。
  • 重量锁:当线程自旋10次(默认可设置),会将轻量锁升级为重量锁,后续线程尝试获取锁时发现时重量锁,会直接挂起,等待系统唤醒。也就是当有一个线程获取锁之后，其余所有等待获取该锁的线程都会处于阻塞状态,会消耗大量的系统资源。
