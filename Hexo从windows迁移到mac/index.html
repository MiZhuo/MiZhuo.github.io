Hexo从windows迁移到mac

发表于2020-01-11|更新于2020-01-11|Hexo
|字数总计:329|阅读时长:1分钟|阅读量:

前言

最近换mac了，所有环境和项目都要转移到Mac上，在迁移Hexo时出了点小问题。这里记录一下。

Step1 配置环境

1、在Mac上安装node和git，这个不用说。用brew两条命令就OK了。

2、配置git

3、配置npm淘宝镜像(加快速度，可省略)

4、使用cnpm

5、生成ssh密钥，关联github、coding

  • 查看是否存在：cd ~/.ssh
  • 生成ssh：ssh-keygen -t rsa -C “youremail@example.com“
  • 输入文件名和密码后打开.pub文件，将公钥配置到github、coding上。
  • 测试是否配置成功：ssh git@github.com

Step2 迁移

1、安装hexo

2、新建目录并初始化Hexo

3、替换文件
将原Hexo中的_config.yml, scaffolds, source, themes替换到本地新建的目录中。

这里如果报TypeError [ERR_INVALID_URL]: Invalid URL:http://at onParseError (internal/url.js:257:9)的错。

就将原来的package.json和package-lock.json也替换过来。把node_modules删除,重新npm install就好了。
又可以愉快的玩耍了。
